#+DATE: 2026-06-17

* Scope
This document is the downstream user contract for packages that consume the
Gerbil Scheme language project harness through =gerbil.pkg= and =gxtest=.

The harness owns policy discovery, reporting, and repair hints. Downstream
projects own their =gerbil.pkg= policy stanza, source-scope roots, runtime roots,
and optional threshold configuration.

* Install Boundary
Install the harness from its checkout into the global Gerbil package store:

#+begin_src shell
gxpkg build
#+end_src

The expected installed command is:

#+begin_src shell
gerbil-scheme-harness guide --downstream
#+end_src

Gerbil package-manager state belongs under =~/.gerbil=. Do not create, depend
on, or commit a repository-local =.gerbil= directory in the downstream project.
If a local package is not visible, fix the global install/load path instead of
adding repository-local package-manager state.

* gerbil.pkg Dependency
Downstream packages should declare the harness dependency in =gerbil.pkg=:

#+begin_src scheme
(package: your/package
 depend: ("github.com/tao3k/gerbil-scheme-language-project-harness"))
#+end_src

Project-specific policy remains package-owned. A downstream package can keep
its own policy stanza, for example:

#+begin_src scheme
(package: your/package
 depend: ("github.com/tao3k/gerbil-scheme-language-project-harness")
 policy: ((source-scope
           roots: (".")
           runtime-roots: ("src")
           explanation: "The harness covers every .ss file in this package.")
          (modularity-policy
           config: "harness-policy/gerbil.ss")))
#+end_src

* gxtest Fixture
Create a small test owner such as =t/project-policy-test.ss=:

#+begin_src scheme
;;; -*- Gerbil -*-
(import :std/test
        :policy/gxtest)

(export project-policy-test)

(def project-policy-test
  (make-policy-test "." ["t/project-policy-test.ss"]))
#+end_src

Run it with:

#+begin_src shell
gxtest t/project-policy-test.ss
#+end_src

On failure, the adapter prints a compact line protocol:

#+begin_example
[gerbil-gxtest] status=fail files=<n> definitions=<n> findings=<n>
|agent-repair-info ...
|finding rule=<rule-id> severity=<severity> path=<path> selector=<selector> message=<message>
|agent-repair ...
|finding-detail ...
#+end_example

* Harness Self-Apply Gates
Ordinary =gxtest= runs should stay close to normal semantic tests. The harness
therefore keeps its full source-policy self-apply gate outside the default test
owner.

Use the fast wiring check during normal development:
#+begin_src shell
gxtest t/self-apply-test.ss
#+end_src

Run the full harness self-apply gate explicitly when auditing source-policy debt:
#+begin_src shell
gxtest t/self-apply-full-gate.ss
#+end_src

The full gate intentionally performs whole-project source analysis and can take
substantially longer than ordinary Gerbil semantic tests. It should fail with a
compact debt summary instead of dumping full snapshot reports.
* Programmatic API
The downstream test surface is:

| API | Use |
|-----+-----|
| =make-gxtest-policy-test= | Default downstream =gxtest= suite entry. Add this to the normal test aggregator so policy runs once with ordinary tests. |
| =make-policy-test= | Minimal =gxtest= suite for the files supplied by the build/test owner. |
| =policy-findings= | Raw =TypeFinding= values for a files-scoped custom test. |
| =policy-status= | =pass= or =fail= status for a files-scoped custom test. |
| =policy-report= | Structured files-scoped report with status, file count, definition count, agent repair, and raw findings. |
| =make-project-policy-test= | Explicit full-project audit suite. Do not use as the default gxtest path. |
| =project-policy-findings= | Raw =TypeFinding= values for custom tests. |
| =project-policy-status= | =pass= or =fail= status for a project root. |
| =project-policy-report= | Structured report with status, file count, definition count, agent repair, and raw findings. |
| =display-project-policy-report= | Compact human-readable report for custom failure paths. |

Use =policy-report= when a downstream package wants custom assertions for the
same file scope that =gxtest= is running. Use =project-policy-report= only for an
explicit full-project audit.

* Agent Contract
Agents integrating the harness downstream should follow this order:

1. Confirm the harness package is globally installed with =gxpkg build= from the
   harness checkout or an equivalent global package install.
2. Add the =gerbil.pkg= dependency without creating a repository-local
   =.gerbil= directory.
3. Add =t/project-policy-test.ss= using =:policy/gxtest=.
4. Run =gxtest t/project-policy-test.ss=.
5. If policy fails, follow the primary =|agent-repair= next command before
   editing downstream source.

The harness CLI remains available for direct checks:

#+begin_src shell
gerbil-scheme-harness check --full .
gerbil-scheme-harness guide --downstream
#+end_src
