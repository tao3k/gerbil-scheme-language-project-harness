* 31.01 Gerbil Checker Roadmap

** 31.01.1 Reference Scope

The local reference checkout at =.data/scheme-harness-checker= shows several
checker capabilities that should inform the Gerbil harness:

- JSON schema projection into checker types.
- Type environment construction from signatures and first-pass constraints.
- Whitelist handling for known terminating collection higher-order functions.
- Termination and PEG walker passes separated from command dispatch.

** 31.01.2 Local Boundary

The Gerbil harness checker starts with a dedicated finding model under
=src/checker/=, while =src/commands/check.ss= remains only a command adapter.
The first rule preserves the existing reader error behavior as
=GERBIL-SCHEME-READ-R001=.

** 31.01.3 Next Rules

The next checker work should add parser-owned facts for imports, declarations,
and call forms before porting higher-level checker rules. The durable order is:

#+begin_src text
parser facts -> checker fact model -> rule implementation -> JSON projection -> tests
#+end_src
