** 11.09.1 Scope
This note turns the Gerbil language-feature audit into a policy and scenario plan.
The goal is not to require advanced syntax everywhere.  The goal is to make high-quality Gerbil idioms visible when parser facts show that they are the right repair boundary.

The first implementation slice covers:
- macro hygiene and thin transformer boundaries
- pattern destructuring through =lambda-match= / =lambda-ematch=
- expression composition through =cut=, =curry=, =rcurry=, =compose=, =rcompose=, =!>=, and =!!>=
- benchmark-owned scenarios for each performance-sensitive policy fixture

** 11.09.3 Feature Classes
*** 11.09.3.1 Macro Hygiene
Quality signal:
- macro syntax is a thin hygienic wrapper
- parsing helpers live in ordinary or =begin-syntax= functions
- runtime behavior remains in ordinary helpers
- docs explain expansion contract and example result

Policy surface:
- Existing owner: =src/policy/agent-style-gerbil-signals.ss=
- Existing facts: =source-file-macros=
- Existing signals: =typed-combinator-style-controlled-macro-syntax-signals=

Scenario requirement:
- input uses a macro-heavy owner without enough controlled helper boundary evidence
- expected keeps macro surface thin and moves runtime behavior into ordinary helpers
- benchmark carries =feature= =macro-hygiene= and the R013 rule id

*** 11.09.3.2 Pattern Destructuring
Quality signal:
- repeated unary destructuring should prefer =lambda-match= or =lambda-ematch=
- a named helper is acceptable only after parser facts rule out a pattern-matching function
- match destructuring in list/table adapters should stay expression-level

Policy surface:
- Existing owner: =src/parser/higher-order.ss=
- Existing roles: =pattern-matching-function= and =lambda-match-opportunity=
- Existing scenario: =controlled-branch-higher-order-performance=

Scenario requirement:
- expected repair must include parser-detectable =lambda-match= or =lambda-ematch=
- benchmark carries =feature= =typed-combinator-style= and =optimizationFocus= that names the branch repair

*** 11.09.3.3 Composition And Pipeline
Quality signal:
- repeated wrapper lambdas should move toward =cut=, =curry=, =rcurry=, =compose=, =rcompose=, =!>=, or =!!>= when arity facts fit
- projection/rendering pipelines should expose selector, line-builder, and reducer boundaries
- hot numeric loops are exceptions only when the source owns a measured hot path

Policy surface:
- Existing owner: =t/policy/agent-style-higher-order-test.ss= for R013 quality repair
- Existing parser roles: =partial-application=, =function-curry=, =function-composition=, and =pipeline-composition=
- Existing source profile: =src/policy/gerbil-utils-source.ss=

Scenario requirement:
- input contains repeated wrapper-lambda drift
- expected contains a composition or specialization idiom
- benchmark records =feature= =higher-order-composition=

*** 11.09.3.4 Hot-Path Exceptions
Quality signal:
- loops and mutation are acceptable for C3 precedence, cache mutation, table state, and numeric hot paths
- these exceptions must carry local comments and benchmark evidence
- style policy must not rewrite a measured hot path into a slower expression pipeline

Policy surface:
- Existing POO performance scenarios already carry benchmark contracts.
- Benchmark contracts expose =hotPathExemption=, =hotPathEvidence=, and =styleRewriteBoundary= so performance exceptions are explicit facts instead of broad policy suppressions.
Scenario requirement:
- every POO performance scenario benchmark declares the exemption kind, evidence categories, and rewrite boundary
- the scenario runner copies those fields into timed receipts so agent-facing diagnostics can cite benchmark evidence without reparsing fixture files

*** 11.09.3.5 Slot Descriptor And Lens Boundary
Quality signal:
- one owner should not mix slot lookup, mutation, modification, and validation in a single broad function
- local slot descriptor or lens helpers should own =get=, =set=, and =modify= symmetry
- validation should stay at the local slot update boundary
- =gerbil-poo/mop.ss= is a learning corpus for this style, not a required dependency

Policy surface:
- Owner: =src/policy/agent-style-gerbil-signals.ss=
- Quality facet: =slot-lens-boundary=
- Agent-facing fields: =slotLensBoundarySignals= and =slotLensBoundaryTargets=
- Reference profile: =src/policy/gerbil-utils-source.ss= exposes =slot-lens-boundary= with examples from =gerbil-poo/mop.ss#slot-checker=, =gerbil-poo/mop.ss#slot-definer=, =gerbil-poo/mop.ss#Lens.modify=, =gerbil-poo/mop.ss#Lens.compose=, and =gerbil-poo/mop.ss#slot-lens=

Scenario requirement:
- input uses a single exported owner whose typed contract collapses =Slot=, =Lens=, =Get=, =Set=, =Modify=, and =Validate=
- expected repair introduces local helpers only; it must not import =gerbil-poo= or =gerbil-utils=
- benchmark carries =feature= =slot-lens-boundary= and the R013 rule id
*** 11.09.3.6 Exception Continuation Boundary
Quality signal:
- one owner should not mix exception capture, continuation capture, contextual logging, and re-raise semantics without a named local boundary
- diagnostics should keep printable exception fallback local
- context logging should happen before re-raise when the operation cannot recover
- =gerbil-utils/exception.ss= is a learning corpus for this style, not a required dependency

Policy surface:
- Owner: =src/policy/agent-style-gerbil-signals.ss=
- Quality facet: =exception-continuation-boundary=
- Agent-facing fields: =exceptionContinuationBoundarySignals= and =exceptionContinuationBoundaryTargets=
- Reference profile: =src/policy/gerbil-utils-source.ss= exposes =exception-continuation-boundary= with examples from =gerbil-utils/exception.ss#string<-exception=, =gerbil-utils/exception.ss#with-catch/cont=, =gerbil-utils/exception.ss#call-with-logged-exceptions=, =gerbil-utils/exception.ss#thunk-with-logged-exceptions=, and =gerbil-utils/exception.ss#with-logged-exceptions=

Scenario requirement:
- input uses a single exported owner whose typed contract collapses =Exception=, =Continuation=, =Handler=, =Context=, and =Raise=
- expected repair introduces local helpers only; it must not import =gerbil-utils=
- benchmark carries =feature= =exception-continuation-boundary= and the R013 rule id

** 11.09.5 Continuing Mining Result
The next quality layer is pinned in the benchmark contract, not in ad hoc policy
suppressions. POO performance fixtures now declare explicit hot-path exemption
metadata:
- =hotPathExemption= names the bounded exception family.
- =hotPathEvidence= lists parser/policy evidence categories such as
  =manual-loop=, =numeric-type-object=, =single-boundary-snapshot=, or
  =benchmark-contract=.
- =styleRewriteBoundary= tells an agent why a generic style rewrite must not
  cross the measured performance boundary.

This keeps the higher-order style policy and the POO performance policy aligned:
agents still prefer expression-level composition by default, but measured POO
hot paths now carry an explicit counter-evidence surface that can be tested.

*** 11.09.6.2 gerbil-utils stateful-avl-map.ss Bounded Mutation
Evidence:
- AVL mutation stays inside the data-structure invariant: height cache maintenance, rotations, balance, put/remove, and copy-node mutation are local.
- Rotation comments document the structural transformation instead of restating code mechanics.
- Mutable traversal is exposed through generator, table, alist, and list-builder adapters at the boundary.

Policy result:
- =gerbil-utils-stateful-structure= is now a source-backed profile.
- The profile distinguishes bounded mutable algorithms from ordinary mutable drift by requiring invariant-local mutation, cached-height maintenance, and adapter boundaries.
*** 11.09.6.3 gerbil-utils generator.ss Control Inversion
Evidence:
- =gerbil-utils/generator.ss= documents generators as dual push/pull interfaces related by inversion of control.
- =generating<-for-each= owns the call/cc =yield= boundary, =unexpected-yield= guard, and out-of-band =eof!= continuation.
- Adapter functions such as =generating<-list= and =list<-generating= keep stream conversion at the boundary.

Policy result:
- =gerbil-utils-generator-control= is now a source-backed profile.
- R013 generator-contract findings select this profile when parser facts expose =generator-combinator-boundary=.
- The profile distinguishes generator control boundaries from ordinary higher-order pipelines by requiring one-shot semantics, yield continuation ownership, and adapter boundaries.
Scenario requirement:
- =t/scenarios/policy/generator-control-performance= owns the input/expected fixture.
- =benchmark.ss= carries =expectedReferencePattern=, =expectedReferenceExamples=, and =expectedQualitySignals= so the scenario checks learned quality evidence instead of relying on a unit-test-only hard-coded expectation.
- The synthetic generator policy test only validates parser facts and steering text; quality reference validation belongs to the scenario gate.
*** 11.09.6.4 gerbil-poo io.ss Serialization Protocol Layering
Evidence:
- =gerbil-poo/io.ss= separates JSON, string, bytes, and marshal protocols instead of collapsing them into one helper.
- =marshal= / =unmarshal= are self-delimited protocol boundaries; =bytes<-= / =<-bytes= are raw representation adapters and do not promise self-delimiting output.
- =methods.string<-json=, =methods.bytes<-marshal=, and =methods.marshal<-bytes= show local method-adapter style.
Policy result:
- =protocol-serialization-boundary= is a neutral quality reference, not a POO extension pattern.
- R013 exposes this reference only when parser-owned typed-contract facts show one owner mixing three or more representation layers such as JSON, String, Bytes, Marshal, or Sexp.
- Agent guidance says to split local protocol adapters and preserve reader/writer symmetry; it explicitly does not require =gerbil-poo= or =gerbil-utils= dependencies.
Scenario requirement:
- =t/scenarios/policy/protocol-serialization-boundary= owns the input/expected fixture.
- The expected fixture uses local helpers only; the reference examples remain learning evidence from =.data/gerbil-poo/io.ss=.
** 11.09.7 gerbil:// Functional Idiom Mining

Source evidence from =gerbil://= shows the repair target should be a
language-shape policy, not a dependency recommendation:

| Source selector | Idiom learned | Policy repair boundary |
|-----------------+---------------+------------------------|
| =gerbil://std/actor-v13/rpc/proto/cipher.ss:130-170= | =foldl= carries the encrypted byte offset through chunks without a hand-written recursion wall. | Warn on pure named-let list accumulators and steer to a reducer boundary. |
| =gerbil://std/actor-v18/executor.ss:100-190= | =cut= expresses a one-argument prefix predicate while keeping the surrounding environment construction data-shaped. | Prefer =cut=/small predicate helpers over repeated anonymous wrapper lambdas. |
| =gerbil://gerbil/compiler/method.ss:420-470= | =ast-case=, =with-syntax=, and =map (cut ... <>)= separate syntax traversal from hygienic reconstruction. | Macro repair should keep syntax wrappers thin and push reusable runtime behavior into helpers. |

The agent-facing repair flow is therefore:

1. Native parser facts classify the redundant source shape, such as manual
   loop drift, repeated destructuring, wrapper-lambda drift, or macro syntax
   scaffolding.
2. Policy emits a bounded warning with one repair move tied to that shape:
   fold/reducer, =cut=/=curry=/pipeline, =match=/=lambda-match= boundary, or
   hygienic macro wrapper.
3. Scenario fixtures lock the before/after source and benchmark contract so
   the warning cannot remain a paper rule.

This section intentionally treats =gerbil://=, =gerbil-utils=, and
=gerbil-poo= as reference corpora. Downstream projects should not be forced to
depend on them just because the policy learned an idiom there.

Policy result:
- =higher-order-expression= now learns from =gerbil://std/actor-v18/executor.ss#cut-prefix-predicate= as a native Gerbil witness for replacing thin wrapper predicates with a bounded =cut= specializer.
- =higher-order-composition-performance= records that source in its scenario benchmark contract through =learnedStyleSources=, =expectedReferenceExamples=, and =expectedQualitySignals= so the repair stays evidence-backed and timed.
- =case-lambda-function-factory= now has its own benchmark contract and scenario gate. It uses =gerbil-utils/base.ss#case-lambda specializers= as the reference-backed repair for wrapper-lambda factories that hide real arity variants, and checks the parser-owned =multi-arity-function= role after repair.

** 11.09.9 gerbil:// Core Match Boundary Mining

Source evidence from =gerbil://gerbil/core/match.ss= adds a native match
mechanism layer to the existing destructuring-combinator scenario. The policy
lesson is quality and performance oriented: use the language's match parser,
expansion-time metadata, and local match extension points when they remove
runtime probing or generated temporary scaffolding. This is not a general
license to introduce broad macro layers.

| Source selector | Idiom learned | Policy repair boundary |
|-----------------+---------------+------------------------|
| =gerbil://gerbil/core/match.ss:100-130= | The parser supports =(apply getf pat)= as applicative destructuring and routes match-specific expansion through =syntax-local-match-macro?=. | Repeated runtime probing can move to a native match/apply boundary when the data shape is the real interface. |
| =gerbil://gerbil/core/match.ss:456-476= | =syntax-local-value= resolves class metadata and accessor lists before generating predicate/body code. | If shape is known at expansion time, prefer syntax-local metadata lookup over repeated runtime discovery. |
| =gerbil://gerbil/core/match.ss:692-712= | =defsyntax-for-match= separates match-specific expansion from a regular macro expander and raises an explicit syntax error when no match definition exists. | Local match extension is allowed when it is narrow, early-failing, and tied to a pattern language boundary. |

Policy result:
- =destructuring-combinator-boundary= now learns from =gerbil://= core
  match, =gerbil-utils=, and =gerbil-poo=.
- The scenario benchmark records =applicative-destructuring-boundary=,
  =syntax-local-match-extension=, =compile-time-metadata-lookup=, and
  =early-syntax-error-boundary= so quality repair does not collapse into
  aesthetic macro use.

** 11.09.10 Non-Goals
- Do not copy =gerbil-utils= or =gerbil-poo= implementation code into the harness.
- Do not require advanced syntax in every file.
- Do not replace measured hot-path loops with aesthetic combinators.
- Do not create new public rule ids before existing R013/R014 surfaces are exhausted.
